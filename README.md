## Project Overview

<b>IUT CafÃ©Cache</b> is a <b>C++ console-based cafeteria management system</b> designed to
digitalize the existing manual token and payment workflow of the IUT cafeteria.
The system focuses on improving speed, transparency, and usability for
<b>non-residential students and teachers</b>.

The project simulates a real-world backend system by handling users, wallets,
e-tokens, menu items, transactions, and administrative controls entirely from scratch.

---

## Motivation

### User-Centric Motivation
- Long queues due to manual token handling  
- Unclear or frequently changing menus  
- Slow and inconvenient recharge process  
- No visibility of spending or purchase history  

### Developer Motivation
- Learn backend system design fundamentals  
- Understand file-based data persistence  
- Practice modular C++ programming without frameworks  
- Prepare for scalable system development in SPL-II  

---

## Requirement Analysis

User requirements were collected through:
- Google Form survey (50+ responses)
- Physical interviews (students, teachers, non-technical users)

### Key User Expectations:
- Digital token generation
- Faster payment workflow
- Wallet balance & recharge system
- Real-time menu visibility
- Spending history & transparency
- Refund / cancellation options
